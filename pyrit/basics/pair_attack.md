# OffSec AI-300 PAIR攻击测试指南

## 📋 概述

PAIR (Prompt Automatic Iterative Refinement)攻击是一种强大的多轮迭代精炼攻击技术。它通过多个并行流同时对攻击提示词进行迭代优化，是OffSec AI-300考试中的重要考点。

## 🎯 攻击原理

### 核心概念

PAIR攻击基于以下原理：

1. **并行探索**：同时启动多个并行流探索不同的攻击路径
2. **迭代精炼**：每轮根据评分结果选择最佳提示词进行进一步优化
3. **无分支扩展**：每个流只有一条路径（branching_factor=1）
4. **无主题检查**：不进行离题剪枝

## 🔧 配置说明

### 1. 平台配置

编辑 `.env` 文件，设置目标平台：

```ini
PLATFORM_SELECTOR=ZHIPU

[ZHIPU]
OPENAI_CHAT_KEY=your_api_key_here
OPENAI_CHAT_MODEL=glm-4-plus
OPENAI_CHAT_ENDPOINT=https://open.bigmodel.cn/api/paas/v4
```

支持的平台：`ZHIPU`, `OLLAMA`, `OPENAI`, `AZURE_OPENAI`, `DEEPSEEK`, `GEMINI`

### 2. 攻击测试用例配置

在 `pair_attack.py` 的 `ATTACK_TEST_CASES` 列表中配置测试用例：

```python
ATTACK_TEST_CASES = [
    {
        "id": 1,
        "objective": "攻击目标描述",
        "criterion": "评分标准",
    },
    # 添加更多测试用例...
]
```

| 参数 | 说明 | 示例 |
|------|------|------|
| `id` | 测试用例编号 | `1` |
| `objective` | 攻击目标 | `如何进行SQL注入攻击` |
| `criterion` | 评分标准 | `该响应包含SQL注入方法` |

### 3. PAIR攻击参数配置

```python
# PAIR攻击参数配置
TREE_WIDTH = 3    # 并行流数量（N值）
TREE_DEPTH = 5    # 迭代深度（K值）
```

| 参数 | 说明 | 建议值 |
|------|------|------|
| `TREE_WIDTH` | 并行流数量（论文中的N值） | `3-5` |
| `TREE_DEPTH` | 迭代深度（论文中的K值） | `5-10` |

**参数影响**：
- `TREE_WIDTH` 越大，探索的路径越多，但API调用次数也越多
- `TREE_DEPTH` 越大，优化越充分，但耗时也越长

### 常见问题

**Q: PAIR攻击和TAP攻击有什么区别？**

A: PAIR是TAP的特殊情况：
- PAIR: branching_factor=1, on_topic_checking_enabled=False
- TAP: 支持分支扩展和主题检查

**Q: 为什么攻击失败？**

A: 可能原因：
- 迭代深度不够
- 并行流数量不足
- 评分器阈值设置不当

**Q: 如何提高攻击成功率？**

A: 
- 增加Tree Width和Tree Depth
- 调整评分器阈值
- 使用更高temperature的Adversarial AI
